## Manual smoke tests before publishing

The renderer can't be fully tested headlessly. Walk this checklist on a clean install before tagging anything:

- [ ] **MCP**: Settings → MCP Servers → Open marketplace. Install GitHub with a PAT. Create a new agent. Ask it `list my open PRs` — agent should use the new MCP tool transparently.
- [ ] **Browser pane**: Open a worker. Click the "Browser" pill. Navigate to two different sites. Switch agents — each remembers its own URL. Click "Send screenshot to agent" — confirm the composer gets `Please read it: <path>`.
- [ ] **Terminal**: Click the "Terminal" pill. Run `git status` interactively. Click "Share last 50 lines with agent" — fenced code block lands in composer. Reload (Ctrl+R) — terminal output is preserved.
- [ ] **Custom slash commands**: Settings → Custom slash commands → Manage. Add `/standup` with template `Quick standup: what did {{selection}} do today?`. Highlight some text. Type `/standup` in composer. Confirm expansion + cursor placement.
- [ ] **Cost budget**: Set a $0.50 budget on an agent. Run an expensive task. See warning at $0.40, modal at $0.50. Click "Raise budget" → $1. Confirm agent continues.
- [ ] **Web cards**: Ask Iris a question that triggers web search. Confirm 3-5 source cards render with favicons + titles + snippets. Click → opens in default browser.
- [ ] **Mica chrome**: Settings → Theme picker → tick "Translucent window". Quit + relaunch. Drag the window over a colorful wallpaper. Sidebar + chat should tint with the wallpaper; code blocks + composer stay opaque.

## Known gaps + follow-ups

1. **Browser pane agent control.** v0.5.0 ships the user-facing pane only; the agent navigates separately via Playwright MCP (installed through the marketplace) in its own Chromium window. A CDP-attached version where the agent drives the embedded pane is queued for v0.5.x.
2. **Cost budget hard pause.** v0.5.1 surfaces the 100% modal but doesn't physically gate the next `sendMessage` in `lib/agent-manager.js` — if the user dismisses the modal and sends again, the message goes through (modal reappears next turn). The `agentBudgetSkipByAgent` flag is already persisted; the agent manager just needs to consult it. ~10 lines of code, deferred to v0.5.3.
3. **node-pty native binding.** First-time contributors must run `npm install` (the postinstall script runs `electron-rebuild -f -w node-pty`) before `npm start` — otherwise the Terminal pill self-disables. Documented in `docs/terminal.md` + `DEPLOY.md`.
4. **Latest.json hashes.** Left as `null` so the user can fill in from real build output. The download page should hide checksums until both fields are non-null.
